# v0.1.3 — Manifest 声明补全 + tls_verify 配置项

**日期**: 2026-06-18

## 修改内容

### manifest.json
- **dependencies**: 添加 `{"name": "slixmpp"}` — 插件强依赖，此前漏声明
- **capabilities**: 添加 `"message_gateway:duplex"` + `"api:adapter.xmpp"` — 如实声明 @MessageGateway 和 @API 公开端点

### config.py + transport.py — 新增 tls_verify 配置项
- `XmppServerConfig` 新增 `tls_verify: bool = Field(default=False)` 字段
- `transport._configure_tls` 根据此值决定证书验证策略：
  - `tls_verify = true` → `ssl.CERT_REQUIRED` + `check_hostname = True`
  - `tls_verify = false`（默认）→ `ssl.CERT_NONE` + `check_hostname = False`（原行为）
- 两种模式下均有清晰的日志输出告知当前安全状态

### config.toml
- 新增 `tls_verify = false` 配置行

### README.md
- 配置示例和文档中反映 `tls_verify` 选项
- 版本历史、修复列表更新
