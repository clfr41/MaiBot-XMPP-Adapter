# Changelog

## 0.1.5-pre (2026-06-20)

### 修复

- **自身消息被自己处理（双重 Bug）**

  - **Bug 1 — MUC 消息双发**：`transport.py` 同时注册了 `"message"` 和 `"groupchat_message"` 两个 slixmpp 事件处理器。对于群聊消息两者同时触发，且 `_on_groupchat_message` 内部调用了 `self._on_message(msg)`，导致每条 MUC 消息被 `_on_payload` 发送两次。Host 端收到两条完全相同的消息，其中一条即为 Bot 自己刚发出的 echo。
    - 修复：删除 `"groupchat_message"` 事件注册及 `_on_groupchat_message` 方法，统一由 `"message"` 事件处理器根据 `msg["type"]` 区分 `chat`/`groupchat`。

  - **Bug 2 — MUC 自身消息检测失效**：`filter_pipeline.py` Step 4 将 `sender_jid`（MUC 昵称如 `"maibot"`）与 `self_id`（bare JID 如 `"bot@example.com"`）比较，类型不同永远不匹配，导致 `ignore_self_message` 对群聊完全不生效。
    - 修复：`run()` 新增 `muc_nickname` 参数。群聊时比较 MUC 昵称，私聊时保持比较 bare JID。`router.py` 从配置提取 MUC 昵称（未配置时回退到 JID 本地部分）并传入 pipeline。

- **登录后获取到已过期的消息**

  - 根因：XMPP 服务器（Prosody/Ejabberd）在 Bot 离线时存储消息，上线后通过 XEP-0203 Delayed Delivery 投递。插件完全不检查 delay 信息，所有消息不分新旧全塞给 Host。
  - 修复：注册 `xep_0203` 插件。`_on_message` 开头检测 `msg["delay"]`，带有 `<delay>` 元素的离线消息直接丢弃（仅记录 debug 日志）。

- **`xmpp.connect()` 返回 False/None 时永久挂起**

  - 根因：TCP/TLS 连接失败时 `slixmpp.ClientXMPP.connect()` 返回 `False`（而非抛出异常），代码未检查返回值，直接 `await xmpp.disconnected`。因连接从未建立，该 future 永远不会 resolve，导致连接循环卡死，重连机制失效。
  - 修复：检查 `connected` 返回值。为 `False`/`None` 时调用 `xmpp.disconnect()` 清理 slixmpp 内部状态，先 `await asyncio.sleep(reconnect_delay)` 再 `continue` 进重连循环。

- **`connect()` 失败后无间隔紧循环**

  - 根因：初次修复用 `continue` 跳过 try 块底部，但 `continue` 直达 while 头，`asyncio.sleep(wait_sec)` 在 try/except/finally 块之外被跳过。日志可见 3 次重试都在同一秒内。
  - 修复：在 `continue` 前显式 `await asyncio.sleep(wait_sec)`。

### Openfire XML stream header 兼容性

- **Openfire 报 "Character is invalid in" XML 解析错误**

  - 现象：slixmpp 连接 Openfire 时，服务端在解析初始 XML stream 阶段即断开（`session: null`），报 `XMLNotWellFormedException: Character is invalid in`。非 TLS 问题（`use_tls=false` 同样故障）。

  - 根因：slixmpp 默认 stream header 包含 `xml:lang='en'` 属性，Openfire 4.7+ (Netty C2S) 的 XML 解码器在某些环境下对此属性敏感，stream 头被拒。

  - 修复：在 `_AdapterXmppClient` 中覆写 `_build_stream_header()`，按标准 XMPP stream header 格式仅保留必要属性（`to`、`xmlns`、`xmlns:stream`、`version`），`xml:lang` 仅在有值时附带。

- **`filter_pipeline.py` 残留误导注释**

  - 将 "Step 4-5 最终缓存" 改为 "过滤管道通过"（已无缓存逻辑，之前重构时移除）。

### 优化

- **`transport.py`**：移除未使用的 `import weakref` 和 `import time`。
