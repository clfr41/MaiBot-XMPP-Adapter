# v0.1.3 — 连接后空消息修复

**日期**: 2026-06-18

## 问题
连接 XMPP 服务器后，总会收到一条空消息送到消息处理器。

## 根因
`transport.py` 中的 `_on_presence_or_iq` 将 presence stanza 和 IQ stanza 当作普通消息送入了 `_on_payload` 消息处理管道。presence/IQ 不是消息类型，不带 body，在上层表现为空消息。

## 修改

### transport.py
- `_on_presence_or_iq` → `_on_non_message_stanza`
- 不再创建 payload 并调用 `transport._on_payload()`，只记录一条 debug 日志后直接返回
- 事件注册同步改为 `_on_non_message_stanza`

### runtime/router.py
- `handle_inbound_message` 开头新增 body 空值守卫：
  ```python
  body = str(payload.get("body") or "").strip()
  if not body:
      self._logger.debug("入站消息 body 为空，已跳过: ...")
      return
  ```
  兜底过滤所有无 body 的入站消息（typing indicator 等）

## 影响范围
- 正常的 `<message/>` 和 `<message type="groupchat"/>` 通过独立的 `_on_message` / `_on_groupchat_message` 事件处理，不受影响
- presence/IQ stanza 不再进入消息管道，空消息消失
